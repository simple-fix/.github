# Contributing to Simple Fix

Most contribution guides read like tax forms. Here's the short version:

## The Rules

1. **Talk first, code second.** Open an issue before diving into a big PR. It saves everyone time.

2. **Keep it simple.** If your change needs a 40-page design doc, it probably needs a rethink.

3. **Sign your work.** By submitting a PR, you certify you have the right to contribute that code under the project's license. No separate sign-off ceremony required.

4. **Be cool.** Read the [Code of Conduct](./CODE_OF_CONDUCT.md). It's short too.

## The Process

1. Fork the repo
2. Create a branch
3. Make your changes
4. Open a PR with a clear description of what and why
5. I'll review it, we'll chat, and if it's good—it ships

No CLA. No committees. Just clean code and good intent.
